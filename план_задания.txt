  1. Геометрия (ellipse.ts)

  computeApex(cx, cy, a, b, theta) — находит правый апекс:
  - Параметрическое уравнение эллипса: x(t) = cx + a·cos(t)·cosθ -
  b·sin(t)·sinθ
  - Апекс там, где dx/dt = 0 → t₀ = atan2(-b·sinθ, a·cosθ)
  - Подставляем t₀ и получаем {x, y}

  computeEllipseFromApex(cx, cy, apexX, apexY, b) — обратная задача (b держим
  константой, это физически оправдано — радиус керна не меняется):

  Когда пользователь тащит апекс мышью — у нас есть новые координаты апекса, и
  нам нужно пересчитать a, b, theta эллипса. Вот для этого она и нужна.
  Без неё мы не можем обновить эллипс при drag апекса.

  Из условий апекса выводятся два уравнения:
  dx² = a²cos²θ + b²sin²θ
  dx·dy = (a²−b²)·sin(2θ)/2
  Решение при фиксированном b:
  θ = atan2(dx·dy, dx² − b²)
  a = sqrt((dx² − b²·sin²θ) / cos²θ)

  2. CoreCanvas.vue

  - SVG 800×220
  - <rect> — фон керна, <line> — ось y=110
  - <ellipse rx=a ry=b transform="rotate(theta, cx, cy)">
  - Два <circle> — хэндлы центра и апекса
  - Drag через pointerdown / pointermove / pointerup на window (надёжнее чем
  mousemove)
  - SVG-координаты через svg.getScreenCTM().inverse() + матрица

  │ Апекс уходит выше/ниже керна    │ apexY зажат в [coreTop+2,           │

  ├─────────────────────────────────┼─────────────────────────────────────┤
  │ Апекс уводят левее центра       │ apexX зажат в cx + minDelta         │
  ├─────────────────────────────────┼─────────────────────────────────────┤

  │ эллипс)                         │                                     │
  ├─────────────────────────────────┼─────────────────────────────────────┤
  │ a < b после пересчёта           │ swap a↔b, theta += 90°              │
  └─────────────────────────────────┴─────────────────────────────────────┘
